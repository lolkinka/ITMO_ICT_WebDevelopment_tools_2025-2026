# Сервис API (FastAPI Gateway)

Основное приложение FastAPI выполняет роль архитектурного шлюза (API Gateway). Оно предоставляет внешний интерфейс для взаимодействия с клиентом, валидирует входящие данные и управляет очередью фоновых задач.

## Асинхронная архитектура эндпоинтов

Внедрение **Celery** позволило перевести ресурсоемкие операции парсинга в неблокирующий режим. При отправке запроса основное API не ожидает окончания скачивания данных с целевого сайта, а мгновенно регистрирует задачу и освобождает поток для обслуживания других пользователей.

---

## Спецификация эндпоинта парсинга

### Вызов асинхронного парсера

* **URL:** `/parse`
* **Метод:** `POST`
* **Заголовки:** `Content-Type: application/json`
* **Статус успешного ответа:** `202 Accepted` (означает, что задача принята в обработку)

#### Тело запроса (Request Body)
В качестве входных данных эндпоинт принимает объект JSON со списком уникальных числовых идентификаторов новостей StopGame:

```json
{
  "article_ids": [55000, 55001, 55002]
}
```

### Успешный ответ (Response 202)
```json
{
  "message": "Parsing task background execution started",
  "task_id": "9e1b21fa-14b8-4e8c-8f3b-67a6d88ffba2",
  "status": "Dispatched"
}
```
