# Архитектура проекта


Проект построен на базе **микросервисной архитектуры** с распределением обязанностей между изолированными Docker-контейнерами.

## Компоненты системы

Система состоит из четырех основных сервисов:

1. **`web_api` (FastAPI)** — шлюз приложения (API Gateway). Принимает запросы от клиентов, отвечает за аутентификацию, бизнес-логику трекера и постановку тяжелых задач в очередь.
2. **`redis` (Брокер)** — высокопроизводительное хранилище данных в оперативной памяти. Используется Celery в качестве брокера сообщений (очереди задач).
3. **`parser` (Celery Воркер)** — изолированный сервис, который слушает очередь задач в Redis. При появлении задачи асинхронно скачивает данные с сайта StopGame и сохраняет их в базу данных.
4. **`db` (PostgreSQL)** — реляционная база данных, хранящая информацию о пользователях, транзакциях и результатах парсинга.

## Схема взаимодействия

```text
[Клиент] 
   │ (HTTP POST /parse)
   ▼
[FastAPI (web_api)] ──(Мгновенный ответ 202 Accepted)──> [Клиент]
   │
   │ (celery.send_task)
   ▼
[Redis (Брокер очереди)]
   ▲
   │ (Подхватывает задачу)
   ▼
[Celery Воркер (parser)]
   │
   ├─(Асинхронные HTTP запросы)─> [StopGame API]
   │
   └─(Запись данных через asyncpg)─> [PostgreSQL (db)]
