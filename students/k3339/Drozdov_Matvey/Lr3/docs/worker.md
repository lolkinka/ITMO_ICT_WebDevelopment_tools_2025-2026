# Фоновый Парсер (Celery Worker)

Сервис `parser` функционирует в режиме изолированного Celery-воркера. Он полностью автономен, не имеет внешних HTTP-портов и взаимодействует с внешним миром исключительно через брокер сообщений Redis и базу данных PostgreSQL.

## Технологический стек и оптимизация
* **Celery** — фреймворк асинхронного выполнения задач.
* **aiohttp** — асинхронный HTTP-клиент, позволяющий выполнять параллельные сетевые запросы без блокировки I/O-потоков воркера.
* **asyncpg** — легковесный и максимально производительный асинхронный драйвер для PostgreSQL, оптимизированный под работу в фоновых процессах с высокой интенсивностью записи.

---

## Логика выполнения задачи `tasks.run_parsing_task`

При поступлении новой задачи из Redis воркер активирует метод, внутри которого инкапсулирована логика извлечения и сохранения данных.

### 1. Управление конкурентностью и защита от блокировок
Для предотвращения перегрузки целевого сервера и защиты от бана по IP-адресу используется асинхронный семафор:
```python
semaphore = asyncio.Semaphore(5)
```

### 2. Алгоритм сбора и обработки данных
Для каждого полученного target_id воркер выполняет следующие шаги:

1. Формирует URL для получения комментариев: https://stopgame.ru/ajax/comments/get?target_id={target_id}&target_type=news
2. Выполняет асинхронный GET-запрос с маскировкой заголовков (HEADERS).
3. Парсит полученный JSON-ответ. Извлекает внутренний массив comments и достает поле userInfo.user_name.
4. Реализует локальный дедупликатор с помощью множества seen_names, исключая повторную обработку одного и того же автора в рамках одной новости.
5. Завершает сбор при достижении лимита: не более 10 уникальных авторов с одной страницы.

### 3. Трансформация данных и хэширование
- Перед записью в базу данных воркер на лету трансформирует сырые данные:
- Обрезает имя пользователя до 50 символов для соответствия лимитам таблицы.
- Генерирует уникальный логин на основе имени в нижнем регистре и случайного четырехзначного числа (например, ivan_4821).
- Вызывает внутреннюю функцию hash_password(), которая хэширует дефолтный пароль "student2026" с использованием безопасного алгоритма (например, bcrypt).

### 4. Асинхронная запись в PostgreSQL
Воркер использует пул соединений `asyncpg.create_pool()`, что позволяет избежать затрат времени на открытие нового TCP-соединения для каждого пользователя. Запись происходит через прямой асинхронный SQL-запрос:

```sql
INSERT INTO "user" (name, login, hashed_password) 
VALUES ($1, $2, $3);
```
